P16 — 歷史事件進度系統 (Incident Progress) 的實作！                            
                                                                                                  
  主要變更如下：                                 
                                                                                                  
  1. 支援「歷史真相 (Truth)」欄位：              
  在事件的 YAML 檔（例如  minxiong_sugar_incident.yaml ）中新增了  truth                          
  屬性。當玩家透過線索解開事件全貌並結案時，將不再只是顯示原本的簡述，而是解鎖隱藏在背後的歷史
  真相。                                                                                          
  2. 進度百分比與視覺化進度條：                                                                   
  改寫了  cmd_incident.c  ( incident  指令)：                                                     
      • 當直接列出所有事件時，會顯示該事件的調查百分比（如  進度: 25% (線索: 1/4) ）。
      • 查看特定事件詳細情報時，不僅會列出已獲取／未獲取的線索，還會渲染一條視覺化進度條（如      
      進度條：[███░░░░░░░] ）。                                                                   
  3. 結案解鎖機制 (Resolve)：                                                                     
  當該事件收集度達到 100%（即所有線索都找齊）時，介面上會提示「證據確鑿 (100%)，可使用            
  incident resolve <事件ID>  解鎖歷史真相」。修改了  incident_d.c  守護進程，在 
  resolve_incident()  結案時，會判斷若 YAML 有設定  truth  就印出真正的歷史真相。                 
  4. 更新了 TODO.md：                                                                             
  將 P16 階段在  docs/TODO.md  中標記為  [x]  已完成。
