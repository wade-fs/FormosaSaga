## 傳統 room.c 的設計假設是：

    一個房間 = 一個固定的地點
    玩家進來、看描述、選出口、離開

##這個假設和《源流福爾摩沙》的世界觀根本衝突。 我需要的是：

    一個地點 = 多個時間層疊的容器
    玩家看見什麼，取決於他理解了多少

## 同一個地方，不同的玩家看見不同的東西

這個機制的技術結構
```
look_settlement(player, settlement_id)
        ↓
1. 基礎描述（所有人都看得見）
        ↓
2. 依序檢查條件層
   condition_layers = [
     { condition: has_footprint("嘉南平原"),
       reveal: NPC老站長 },

     { condition: has_footprint("西來庵"),
       reveal: "空氣裡有什麼東西" },

     { condition: has_talent("island_memory")
                  AND era_completed("v2.1"),
       reveal: 歷史層入口 },
   ]
        ↓
3. 組合輸出
```

條件可以是任意組合：

. 踏印類型與數量
. 完成的歷史事件
. 職涯等級
. 時間（遊戲內時間、節慶）
. 全服狀態（其他玩家觸發了某事件）
. 聚落記憶完整度本身

## 最重要的設計原則
	- 玩家不知道自己「還沒看見什麼」。
	- 不會有提示說「你還有3個條件未解鎖」。
	- 只有當條件達到，東西自然浮現。
	- 就像真實的理解一樣：
	- 你不知道你不知道什麼。
	- 直到有一天，你突然看見了。
